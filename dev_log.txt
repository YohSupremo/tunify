HOW TO VERIFY / TEST
  STEP 1 — Start the backend server
           > Open a terminal inside tunify_backend/
           > Run: node app.js
           > Expected: "Connected Successfully 5000" printed in console

  STEP 2 — Log in as admin via the frontend
           > Open browser: http://localhost/tunify/login.html
           > Credentials: admin@tunify.com / password123
           > Expected: Redirected to dashboard, token saved in sessionStorage

  STEP 3 — Navigate to Customers page
           > Click "Customers" in the admin navbar
             OR go to: http://localhost/tunify/admin/customers.html
           > Expected: DataTable loads with registered customer rows
                       Columns visible: ID, Name, Email, Phone,
                       Orders, Addresses, Joined, Actions

  STEP 4 — Test READ (GET /api/v1/customers)
           > Open browser DevTools > Network tab
           > Refresh the Customers page
           > Expected: XHR request to /api/v1/customers returns 200 with JSON array
                       Each row has: user_id, email, first_name, last_name,
                       phone, order_count, address_count, created_at

  STEP 5 — Test UPDATE (PUT /api/v1/customers/:id)
           > Click the Edit (pencil) icon on any customer row
           > Expected: Modal opens with correct pre-filled values
                       Address sub-panel shows customer's saved addresses
           > Change First Name or Phone > click Save Customer
           > Expected: SweetAlert2 "Customer updated!" toast appears
                       Table reloads with updated values
           > Verify in DB: SELECT * FROM customer WHERE user_id = <id>;
                           SELECT * FROM users WHERE id = <id>;

  STEP 6 — Test DEACTIVATE (DELETE /api/v1/customers/:id)
           > Click the Deactivate (user-slash) icon on a customer row
           > Expected: Bootbox confirm dialog appears
           > Click "Yes, Deactivate"
           > Expected: SweetAlert2 success toast, customer disappears from table
           > Verify in DB: SELECT deleted_at FROM users WHERE id = <id>;
                           Should show a timestamp (NOT NULL)
           > Verify guard: Try deactivating user_id=1 (admin) via Postman:
                           DELETE http://localhost:5000/api/v1/customers/1
                           Header: Authorization: Bearer <token>
                           Expected: 403 "Cannot deactivate an admin account"

  STEP 7 — Test AUTH PROTECTION
           > Open Postman (or browser DevTools)
           > Call GET http://localhost:5000/api/v1/customers WITHOUT a token
           > Expected: 401 "Login first to access this resource"

  STEP 8 — Test NAVBAR active state
           > On the Customers page, check the navbar
           > Expected: "Customers" link is highlighted/active
                       All other links are not active

           - Addresses not loading in modal → Check GET /api/v1/customers/:id
